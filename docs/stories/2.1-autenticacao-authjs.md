# Story 2.1 — Autenticacao com Auth.js v5 (Google + GitHub)

## Descricao
Como usuario, preciso fazer login com minha conta Google ou GitHub de forma rapida e sem atrito, para poder criar paletas e salvar favoritos.

## Criterios de Aceitacao
- [ ] Auth.js v5 instalado e configurado para Edge Runtime
- [ ] DrizzleAdapter conectado ao D1
- [ ] Login com Google OAuth funcionando
- [ ] Login com GitHub OAuth funcionando
- [ ] Sessao persistida no D1 (tabelas accounts + sessions)
- [ ] Callback de sessao retorna user.id e user.role
- [ ] Botao de login no header (discreto, a direita)
- [ ] Botao de logout no dropdown do usuario
- [ ] Middleware protege rotas /create e /collections
- [ ] Rota /admin protegida por role === 'admin'

## Tarefas
- [ ] `npm install next-auth@beta @auth/drizzle-adapter`
- [ ] Criar `src/auth.ts` com configuracao NextAuth
- [ ] Criar `src/app/api/auth/[...nextauth]/route.ts`
- [ ] Configurar Google OAuth (console.cloud.google.com)
- [ ] Configurar GitHub OAuth (github.com/settings/developers)
- [ ] Adicionar credentials em `.dev.vars`
- [ ] Criar componente `src/components/auth/user-button.tsx` (avatar + dropdown)
- [ ] Criar componente `src/components/auth/sign-in-button.tsx`
- [ ] Atualizar header com botao de login/usuario
- [ ] Atualizar middleware.ts para proteger rotas autenticadas
- [ ] Testar fluxo completo de login/logout

## Dependencias
- Story 1.3 (schema do banco com tabelas auth)
- Story 1.4 (header para colocar botao)

## Notas Tecnicas
- Auth.js v5 e Edge-compatible
- DrizzleAdapter conecta diretamente nas tabelas users/accounts/sessions do schema
- AUTH_SECRET necessario: `openssl rand -base64 32`
- Google OAuth: criar projeto no Google Cloud Console, habilitar Google+ API
- GitHub OAuth: Settings > Developer Settings > OAuth Apps
