# Story 7.1 — Internacionalizacao (i18n) com Rotas

## Descricao
Como visitante internacional, preciso acessar o site no meu idioma (ingles, portugues ou espanhol), para ter uma experiencia completa e o Google indexar conteudo em cada idioma separadamente.

## Criterios de Aceitacao
- [ ] Rotas com prefixo de locale: `/en`, `/pt`, `/es`
- [ ] Middleware detecta idioma do navegador e redireciona
- [ ] `<link rel="alternate" hreflang="...">` em todas as paginas
- [ ] Strings da UI traduzidas nos 3 idiomas
- [ ] Conteudo SEO das paletas servido no idioma correto (tabela palette_content)
- [ ] Seletor de idioma discreto no footer
- [ ] URL sem locale redireciona para idioma detectado

## Tarefas
- [ ] Reestruturar rotas para `app/[locale]/...`
- [ ] Criar `src/middleware.ts` com deteccao de idioma
- [ ] Criar `src/lib/i18n.ts` com dicionarios de traducao
- [ ] Criar dicionarios: `src/messages/en.json`, `pt.json`, `es.json`
- [ ] Implementar `generateMetadata()` com hreflang alternates
- [ ] Criar componente seletor de idioma no footer
- [ ] Atualizar todas as paginas para usar traducoes
- [ ] Atualizar queries de palette_content para filtrar por locale

## Dependencias
- Story 1.4 (layout base)
- Story 3.2 (pagina da paleta com conteudo)

## Notas Tecnicas
- Abordagem: route-based i18n (melhor para SEO que cookie-based)
- Middleware: detectar Accept-Language header, redirect se sem locale
- Dicionarios simples (JSON) — sem lib externa pesada
- Conteudo SEO ja esta no banco por locale (palette_content.locale)
- Default: /en para buscas globais

## Arquivos Afetados
- `src/app/[locale]/layout.tsx`
- `src/app/[locale]/page.tsx`
- `src/app/[locale]/palette/[slug]/page.tsx`
- `src/middleware.ts`
- `src/lib/i18n.ts`
- `src/messages/en.json`
- `src/messages/pt.json`
- `src/messages/es.json`
- `src/components/layout/footer.tsx` (seletor de idioma)
