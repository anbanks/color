# Story 8.1 — Geracao de Conteudo com OpenAI

## Descricao
Como admin, ao aprovar uma paleta preciso que a IA gere automaticamente titulo, descricao, aplicacoes praticas e psicologia das cores nos 3 idiomas, para ter conteudo SEO rico sem esforco manual.

## Criterios de Aceitacao
- [ ] Endpoint `POST /api/generate` que recebe paletteId e gera conteudo
- [ ] Gera conteudo em 3 idiomas (en, pt, es) em uma unica chamada
- [ ] Salva na tabela palette_content (1 registro por idioma)
- [ ] Conteudo gerado: title, description, applications, psychology
- [ ] Usa gpt-4o-mini para custo baixo
- [ ] Protegido: apenas admin pode chamar
- [ ] Nao regera conteudo se ja existe (idempotente)
- [ ] Loading state no painel admin durante geracao

## Tarefas
- [ ] `npm install openai`
- [ ] Criar `src/lib/generate-content.ts` com funcao de geracao
- [ ] Criar `src/app/api/generate/route.ts` (POST, admin only)
- [ ] Prompt engineering: gerar JSON com titulo, descricao, aplicacoes, psicologia
- [ ] Gerar para 3 idiomas (batch ou 3 chamadas paralelas)
- [ ] Salvar resultados na tabela palette_content
- [ ] Integrar com fluxo de aprovacao no admin (Story 5.1)
- [ ] Adicionar OPENAI_API_KEY no .dev.vars
- [ ] Tratamento de erro: retry 1x se falhar

## Dependencias
- Story 5.1 (painel admin — trigger de aprovacao)
- Story 1.3 (tabela palette_content)

## Notas Tecnicas
- Modelo: gpt-4o-mini (rapido, barato, bom para textos curtos)
- Response format: JSON mode (`response_format: { type: 'json_object' }`)
- Custo estimado: ~$0.001 por paleta (3 idiomas)
- Prompt deve gerar conteudo unico e de qualidade para evitar penalidade SEO
- Considerar gerar os 3 idiomas em 1 prompt para economizar tokens

## Arquivos Afetados
- `src/lib/generate-content.ts`
- `src/app/api/generate/route.ts`
