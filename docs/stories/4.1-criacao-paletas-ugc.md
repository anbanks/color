# Story 4.1 — Criacao de Paletas (UGC)

## Descricao
Como usuario logado, preciso criar minhas proprias paletas de 4 cores usando um color picker intuitivo, para contribuir com a comunidade e ver minha paleta publicada.

## Criterios de Aceitacao
- [ ] Rota `/create` protegida por autenticacao
- [ ] Interface com 4 color pickers lado a lado
- [ ] Preview em tempo real da paleta sendo criada
- [ ] Input de tags (max 5 tags)
- [ ] Botao "Submit" envia paleta com status "pending"
- [ ] Feedback visual: "Paleta enviada para moderacao!"
- [ ] Gerar slug automaticamente a partir das cores
- [ ] Validacao: 4 cores obrigatorias, formato HEX valido

## Tarefas
- [ ] Criar `src/app/create/page.tsx` (protegida)
- [ ] Criar `src/components/palette/palette-creator.tsx`
- [ ] Implementar color picker (input type="color" + input HEX manual)
- [ ] Criar preview com PaletteCard em tempo real
- [ ] Adicionar input de tags com autocomplete
- [ ] Implementar API `POST /api/palettes` (cria com status pending)
- [ ] Criar `src/lib/generate-slug.ts` (cores para slug)
- [ ] Validacao client-side e server-side
- [ ] Redirect para home apos submit com toast de sucesso

## Dependencias
- Story 2.1 (autenticacao)
- Story 3.1 (componente PaletteCard para preview)

## Notas Tecnicas
- Color picker: combinar input[type=color] nativo com input text HEX
- Slug: remover # dos HEX e concatenar com hifen (ff5733-33ff57-3357ff-f3ff33)
- Tags sugeridas: warm, cool, pastel, vibrant, dark, earth, sunset, ocean, forest, neon
- Server Action ou API route para submit
- Rate limit: max 10 paletas por dia por usuario
