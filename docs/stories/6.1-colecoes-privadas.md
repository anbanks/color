# Story 6.1 — Colecoes Privadas (Favoritos na Nuvem)

## Descricao
Como usuario logado, preciso salvar paletas em colecoes privadas no meu dashboard, para organizar minhas paletas favoritas sem depender de LocalStorage.

## Criterios de Aceitacao
- [ ] Rota `/collections` protegida por autenticacao
- [ ] Criar nova colecao com nome customizado
- [ ] Adicionar paleta a uma colecao (botao no card da paleta)
- [ ] Remover paleta de uma colecao
- [ ] Visualizar colecoes com grid de paletas
- [ ] Deletar colecao inteira
- [ ] Colecao default "Favoritos" criada automaticamente no primeiro save

## Tarefas
- [ ] Criar `src/app/collections/page.tsx` (lista de colecoes)
- [ ] Criar `src/components/palette/save-button.tsx` (botao salvar no card)
- [ ] Criar dialog de selecao de colecao (ou criar nova)
- [ ] Implementar API `GET /api/collections` (listar)
- [ ] Implementar API `POST /api/collections` (criar)
- [ ] Implementar API `PATCH /api/collections/[id]` (add/remove paleta)
- [ ] Implementar API `DELETE /api/collections/[id]` (deletar)
- [ ] Atualizar PaletteCard com botao de salvar
- [ ] Grid de paletas dentro de cada colecao

## Dependencias
- Story 2.1 (autenticacao)
- Story 3.1 (PaletteCard)

## Notas Tecnicas
- Tabelas: collections + collection_palettes (many-to-many)
- Botao salvar: icone bookmark, dialog com lista de colecoes + "Nova colecao"
- Colecao default criada via trigger na primeira vez que usuario salva
