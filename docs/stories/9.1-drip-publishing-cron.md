# Story 9.1 — Drip Publishing (Cron Triggers)

## Descricao
Como estrategia de SEO, preciso que paletas aprovadas sejam publicadas gradualmente (~5 a cada 4 horas) via Cloudflare Cron Triggers, para que o Google perceba o site como constantemente atualizado.

## Criterios de Aceitacao
- [ ] Cron Trigger configurado no wrangler.toml (a cada 4 horas)
- [ ] Endpoint `/api/cron/publish` protegido por secret
- [ ] Busca ate 5 paletas com status "approved" (mais antigas primeiro)
- [ ] Atualiza status para "published" e define publishedAt
- [ ] Invalida cache KV (trending, popular)
- [ ] Log de quantas paletas foram publicadas
- [ ] Nao faz nada se nao ha paletas aprovadas na fila

## Tarefas
- [ ] Atualizar `wrangler.toml` com cron trigger: `0 */4 * * *`
- [ ] Criar `src/app/api/cron/publish/route.ts`
- [ ] Verificar autenticacao via header (CRON_SECRET)
- [ ] Query paletas approved ordenadas por createdAt ASC, limit 5
- [ ] Update batch para status published + publishedAt = now
- [ ] Invalidar chaves KV: palettes:trending, palettes:popular
- [ ] Adicionar CRON_SECRET no .dev.vars
- [ ] Testar localmente com curl

## Dependencias
- Story 5.1 (paletas com status approved)
- Story 1.1 (wrangler.toml com KV binding)

## Notas Tecnicas
- Cron Triggers sao nativos do Cloudflare Workers
- A cada 4h = ~30 paletas/dia se houver estoque
- Autenticacao: comparar header Authorization com CRON_SECRET
- KV invalidation: delete keys de cache para forcar refresh
- Em dev local, testar com: `curl -H "Authorization: Bearer secret" http://localhost:3000/api/cron/publish`

## Arquivos Afetados
- `wrangler.toml` (adicionar cron)
- `src/app/api/cron/publish/route.ts`
- `.dev.vars` (CRON_SECRET)
