# Story 3.2 — Pagina Individual da Paleta (SEO)

## Descricao
Como visitante vindo do Google, preciso ver uma pagina dedicada para cada paleta com interface limpa acima da dobra e conteudo rico abaixo, para copiar cores rapidamente e o Google indexar conteudo de qualidade.

## Criterios de Aceitacao
- [ ] Rota `/palette/[slug]` com pagina dedicada
- [ ] Acima da dobra: paleta grande com cores expandidas + botoes de copia (HEX, RGB, HSL)
- [ ] Abaixo da dobra: titulo, descricao, aplicacoes praticas, psicologia das cores
- [ ] Metadata dinamica (title, description, og:image)
- [ ] Schema.org structured data (CreativeWork)
- [ ] Botao de like e compartilhar
- [ ] Paletas relacionadas no final
- [ ] ISR com revalidate de 1 hora
- [ ] Fallback para paletas sem conteudo gerado

## Tarefas
- [ ] Criar `src/app/palette/[slug]/page.tsx`
- [ ] Criar `src/components/palette/palette-detail.tsx` (visualizacao grande)
- [ ] Criar `src/components/palette/copy-button.tsx` (HEX/RGB/HSL toggle)
- [ ] Criar `src/components/seo/palette-content.tsx` (conteudo abaixo da dobra)
- [ ] Implementar `generateMetadata()` com dados da paleta
- [ ] Adicionar JSON-LD Schema.org (CreativeWork)
- [ ] Implementar API `GET /api/palettes/[id]` para detalhe
- [ ] Query paletas relacionadas (mesmas tags ou cores similares)
- [ ] Configurar ISR: `revalidate = 3600`

## Dependencias
- Story 3.1 (componentes de paleta)
- Story 1.3 (tabela palette_content)

## Notas Tecnicas
- Slug: gerado a partir das cores (ex: ff5733-33ff57-3357ff-f3ff33)
- Acima da dobra: ZERO texto, apenas cores e botoes — UX pura
- Abaixo da dobra: conteudo SEO rico, pode ter ~500 palavras
- Structured data: @type CreativeWork, name, description, creator, datePublished
- OG Image: gerar imagem com as 4 cores (futura story com R2)
