# Story 5.1 — Painel Admin (Moderacao de Paletas)

## Descricao
Como admin, preciso de um painel para aprovar ou rejeitar paletas submetidas pelos usuarios, para manter a qualidade do conteudo publicado.

## Criterios de Aceitacao
- [ ] Rota `/admin` protegida por role === 'admin'
- [ ] Dashboard com contadores: pendentes, aprovadas, rejeitadas, publicadas
- [ ] Lista de paletas pendentes com preview visual
- [ ] Botoes Aprovar / Rejeitar em cada paleta
- [ ] Ao aprovar: status muda para "approved" e dispara geracao de conteudo IA
- [ ] Ao rejeitar: status muda para "rejected"
- [ ] Filtro por status (pending, approved, rejected, published)
- [ ] Paginacao na lista

## Tarefas
- [ ] Criar `src/app/admin/layout.tsx` (verificar role admin)
- [ ] Criar `src/app/admin/page.tsx` (dashboard)
- [ ] Criar `src/app/admin/palettes/page.tsx` (lista de moderacao)
- [ ] Criar `src/components/admin/moderation-card.tsx`
- [ ] Criar `src/components/admin/moderation-list.tsx`
- [ ] Criar `src/components/admin/stats-cards.tsx` (contadores)
- [ ] Implementar API `PATCH /api/palettes/[id]` (aprovar/rejeitar)
- [ ] Ao aprovar: chamar endpoint de geracao de conteudo IA
- [ ] Paginacao com query params (?page=1&status=pending)

## Dependencias
- Story 2.1 (autenticacao com roles)
- Story 4.1 (paletas com status pending)

## Notas Tecnicas
- Admin check: `session.user.role === 'admin'`
- Primeiro usuario pode ser promovido a admin via SQL direto no D1
- Geracao de conteudo IA pode ser assincrona (nao bloquear a UI)
- Usar Server Actions para aprovar/rejeitar (mais rapido que API routes)
