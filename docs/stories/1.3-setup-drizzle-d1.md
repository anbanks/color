# Story 1.3 — Configurar Drizzle ORM + Cloudflare D1 + Schema

## Descricao
Como desenvolvedor, preciso configurar o Drizzle ORM com Cloudflare D1 e criar o schema completo do banco de dados, para ter persistencia de dados na borda com type-safety.

## Criterios de Aceitacao
- [ ] Drizzle ORM instalado e configurado para D1
- [ ] `drizzle.config.ts` criado
- [ ] Schema completo criado com todas as tabelas
- [ ] Migration inicial gerada via `drizzle-kit generate`
- [ ] Migration aplicada localmente via wrangler
- [ ] Helper `getDb()` criado para instanciar conexao
- [ ] Seed script com 10 paletas de exemplo

## Tarefas
- [ ] `npm install drizzle-orm` e `npm install -D drizzle-kit`
- [ ] Criar `drizzle.config.ts`
- [ ] Criar `src/db/schema.ts` com tabelas: users, accounts, sessions, palettes, palette_content, likes, collections, collection_palettes
- [ ] Criar `src/db/index.ts` com helper getDb()
- [ ] `npx drizzle-kit generate` para gerar migration
- [ ] `npx wrangler d1 create color-db` para criar banco local
- [ ] `npx wrangler d1 migrations apply color-db --local`
- [ ] Criar `src/db/seed.ts` com paletas de exemplo
- [ ] Testar queries basicas

## Dependencias
- Story 1.1 (wrangler configurado)

## Notas Tecnicas
- D1 e SQLite — usar tipos SQLite (text, integer, real)
- IDs como text (cuid2) — `npm install @paralleldrive/cuid2`
- Timestamps como integer com mode: 'timestamp'
- colors e tags armazenados como JSON string
- Drizzle e o unico ORM compativel com Edge Runtime + D1

## Schema (Tabelas)
1. `users` — id, name, email, image, role, createdAt
2. `accounts` — id, userId, type, provider, providerAccountId, tokens
3. `sessions` — id, userId, sessionToken, expires
4. `palettes` — id, slug, colors, tags, status, authorId, likesCount, publishedAt, createdAt
5. `palette_content` — id, paletteId, locale, title, description, applications, psychology, createdAt
6. `likes` — id, userId, paletteId, createdAt
7. `collections` — id, name, userId, createdAt
8. `collection_palettes` — id, collectionId, paletteId, addedAt
